**Why:** Two pressures converged. (1) 2026-04-26 Anthropic credits exhausted + 2026-04-27 DNS blip lost 2,705 papers — cloud-LLM single-points-of-failure. (2) 10k MIT OCW papers being ingested across math/physics/bio/CS/anthropology/etc. for a multistrategy fund (quantamental + social signals); cloud-only LLM at 25k corpus = ~$300-500/mo, local-first = ~$70/mo.

**How to apply:**
- **Resilience:** when an Anthropic / Gemini call surfaces a rate-limit / credits-exhausted error mid-run, the failover wrapper in `python/llm/client.py:complete()` automatically retries via OpenRouter if `OPENROUTER_API_KEY` is set. `BudgetExceeded` (our own circuit breaker) is NEVER retried.
- **Cost:** flip `OLLAMA_LOCAL=1` in env to swap paper_triage flash/deepseek tiers + factor_extractor default + structural_break default to local llama3.2:3b / qwen2.5:7b-instruct. Sonnet/Opus tiers stay cloud (reasoning-heavy work where alpha lives). Calibration drill REQUIRED before flipping in prod: `ollama pull llama3.2:3b qwen2.5:7b-instruct; OLLAMA_LOCAL=1 python -m python.research.papers.paper_triage --limit 100`; ship if tier-distribution agreement vs Gemini-flash ≥70%.
- **Cross-disciplinary substrate:** `kb_sources.domain` + `kb_concepts.source_domain` + `kb_concept_links.relation='analog_of'` (with `analogy_strength` + `analogy_rationale`) are now first-class. Each strategy's `program.md` should declare `ideation_domains: [...]` so autoresearch RAG retrieval scopes to the declared domains. The `Corpus` protocol in `python/research/knowledge/corpus.py` formalizes the 9-harvester pattern; new harvesters implement it.
- **OCW ingest closed end-to-end (2026-05-02):** `harvest_mit_ocw.py` has `fetch_search_results(base_url)` (Firecrawl `/v2/scrape` of the OCW search page → markdown+links → filter to OCW resource URLs → `OcwSearchHit[]`; topics inherited from search URL's `t=` facets, course_id + resource_kind URL-derived) AND `fetch_pdf_text(url)` (Firecrawl `/v2/scrape` for the per-PDF parse, returns markdown). Both fail closed (never raise). Note on Firecrawl endpoints: `/v2/parse` is local-file multipart only; `/v2/scrape` is the right call for remote PDF URLs. Cost envelope: 10k OCW resources × ~8 pages = ~80k Firecrawl credits — plan against quota.
- **Cited web-research loop (Phase H, 2026-05-02):** `python/research/knowledge/web_research.py` ports the [firesearch](https://github.com/firecrawl/firesearch) pattern (decompose → search × N → validate ≥0.7 conf → retry rephrased → synthesize with citations → follow-ups) into Python. All LLM calls go through `python.llm.complete()` so they inherit cost ledger + OpenRouter failover + `OLLAMA_LOCAL=1`. Light passes (decompose / validate / follow-ups) use Haiku-or-llama3.2:3b; synthesis uses Sonnet 4.6. Output: `ResearchReport` with `report_to_markdown()` for `outputs/autoresearch/web_research/<slug>.md`. Tunables match firesearch defaults: MAX_SUB_QUESTIONS=4, MIN_CONFIDENCE=0.7, MAX_RETRIES=1, SEARCH_K=5. Cost envelope per question: ~$0.05 cloud-Sonnet synthesis + ~$0.005 fully-local light passes + 5 search + 5 scrape Firecrawl credits.
- **Karpathy autoresearch wire-in (2026-05-02):** `python/research/knowledge/strategy_research.py` exposes `refresh(strategy_name)` (operator-driven CLI) that derives a research question from `strategies/<name>/program.md` (priority: explicit `research_question:` YAML field → first thesis paragraph → generic fallback), runs `web_research.run()`, writes a sentinel-headered cached note to `strategies/<name>/web_research.md`. `autoresearch/variation_runner.py:_call_claude` now calls `_load_web_research(name)` and prepends the cached note as a `=== Recent web research ===` block in the prompt — backward-compat: returns "" when cache absent, no behavior change for unrefreshed strategies. **Cost decoupling:** refresh cadence is operator's choice (~$0.05/strategy/refresh × 50 strategies × daily = ~$2.50/day); variation iteration stays free of network calls. CLI: `python -m python.research.knowledge.strategy_research --strategy <name>` or `--all`. Schedule via `/schedule` for daily refresh once founder is ready. Tests: 17 in `test_strategy_research.py`; 37 existing autoresearch tests still green after the variation_runner injection.
