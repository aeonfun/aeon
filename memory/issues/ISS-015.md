status: open

affected_skills:

root_cause: |
  Line 577 reads `MESSAGE=$(echo '${{ toJson(github.event.client_payload.message) }}' | jq -r '.')`.
  `toJson()` produces a JSON-encoded string but does NOT escape single quotes
  (single quotes are not special in JSON strings). When the JSON string contains
  a literal `'`, it closes the bash single-quoted region and the rest of the
  payload is parsed as shell. A dispatcher posting a payload like
  `'; touch /tmp/pwn; '` breaks out into command execution.

  This is the canonical incident pattern flagged in
  `articles/workflow-security-audit-2026-04-11.md`. The 2026-04-11 fix rebound
  `inputs.message` to `_INPUT_MESSAGE` correctly but missed the
  `client_payload.message` branch.

fix_pr: null

## Remediation
Rebind to an env key, read the env var via standard double-quoted shell:

```yaml
env:
  _DISPATCH_PAYLOAD: ${{ toJson(github.event.client_payload.message) }}
run: |
  MESSAGE=$(echo "$_DISPATCH_PAYLOAD" | jq -r '.')
```

Env-var values are not subject to template-string interpolation, so the bash
quoting is safe regardless of payload contents.
