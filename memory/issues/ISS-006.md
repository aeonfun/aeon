# ISS-006 — monitor-runners artifact overwritten to `--help`

## Summary

On the 2026-05-20 morning-review chain run, monitor-runners executed cleanly and
the skill log records a 1.5kb v2-locked signal written to
`.outputs/monitor-runners.md`. The on-disk file holds the literal seven-byte
payload `--help\n`. The chain context passed to daily-ops-review (chain Step 4)
also shows the monitor-runners section as `--help`, so the overwrite occurred
between the skill's write and Step 4's read.

## Impact

- High. monitor-runners output is lost for this chain run. perps-brief and
  morning-macro consumed an empty section in place of the DEEP-LIQ verdict, and
  daily-ops-review cannot audit the runner-side signal.
- Different failure mode from ISS-005. The artifact is not the wrong format. It
  is replaced wholesale by an unrelated string.

## Hypotheses

- A `--help` invocation against the skill runner or notify script landed in the
  wrong place and shell-redirected over the artifact.
- A workflow step in `chain-runner.yml` or a postprocess script ran with an
  argument-parse path that wrote `--help` to stdout and clobbered the file.
- An aliased command in the sandbox printed its usage banner to the artifact path
  via a stray `>` redirect.

## Recommended next step

Inspect `chain-runner.yml` and `scripts/postprocess-*.sh` for any step that could
write to `.outputs/monitor-runners.md` after the skill exits. Check whether the
monitor-runners skill itself has a `--help` branch and whether a workflow step
shells the skill name with `--help` for documentation purposes.
