## Reproduction

1. From the GitHub Actions sandbox, run the skill's curl as documented:
   ```
   UA="web:aeon-vibecoding-digest:1.0 (by /u/aeonbot)"
   curl -fsSL -H "User-Agent: $UA" "https://old.reddit.com/r/vibecoding/top.json?t=day&limit=30"
   ```
   → HTTP 403, body is the Reddit "Blocked" interstitial.
2. Same with `https://www.reddit.com/...` → HTTP 403.
3. WebFetch on either URL → "Claude Code is unable to fetch from ...".
4. UA variations (Mozilla-style, `aeon/1.0`, etc.) all 403.

## What needs to happen

Pick one:

**Option A — authenticated Reddit API via prefetch (recommended).** Create
`scripts/prefetch-reddit.sh` that runs **before** Claude starts (full network access from the
GHA runner) and:

- Reads `REDDIT_CLIENT_ID` / `REDDIT_CLIENT_SECRET` from secrets, exchanges for an OAuth token
  against `https://www.reddit.com/api/v1/access_token` with grant `client_credentials`.
- Calls `https://oauth.reddit.com/r/vibecoding/{top,hot,rising}.json` with the bearer token
  and the same UA. The OAuth API typically does not block GHA IPs the way the public endpoint
  does, because authenticated requests are billed against the app's quota.
- Writes the three JSON payloads to `.reddit-cache/vibecoding/{top,hot,rising}.json`.
- The skill's step 1 reads from `.reddit-cache/vibecoding/*.json` instead of curling Reddit
  directly. Comments fetch (step 4) follows the same pattern: prefetch script also writes
  `.reddit-cache/vibecoding/comments-{post_id}.json` for top-N posts, OR the skill skips
  comments fetches when uncached and degrades the spicy-threads section gracefully.

**Option B — third-party mirror.** If Pushshift or a community mirror is back online, point
the skill at it. Lower reliability than option A.

**Option C — disable until path is in place.** Remove the cron entry and stop firing the skill
until either prefetch script ships, to avoid daily error spam in `./notify`.

## Notes

- The skill's SKILL.md already documents the WebFetch fallback as the sandbox workaround;
  that section is wrong for Reddit specifically and should be amended once the prefetch is in.
- `.reddit-cache/` directory will need to exist as a tracked empty dir (or `.gitkeep`) so
  Claude can write to it without hitting "create directory" deny — same gotcha noted in
  ISS-001 for `.vuln-scan/`.
- Same pattern exists in `scripts/prefetch-xai.sh` (only existing prefetch script today) —
  use it as the template.
- This is the second sandbox-limitation issue filed today (after ISS-001 vuln-scanner) with
  the same shape: skill needs a network-fetch step that must run pre-sandbox. Worth tracking
  as a class of problem.
