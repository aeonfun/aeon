# ISS-001 — perps-scan cannot fetch Coinglass v4

## Summary

The Coinglass migration of `perps-scan` (v2.1) replaced the geo-blocked Bybit data
layer with Coinglass v4. Coinglass v4 is reachable but requires an authenticated
header. In the GitHub Actions sandbox the skill's documented inline-curl approach
fails for two compounding reasons:

1. **Inline env-var expansion is blocked.** `curl -H "CG-API-KEY: $COINGLASS_API_KEY"`
   is rejected by sandbox static analysis ("Contains expansion") — a deliberate
   anti-exfiltration measure.
2. **Script-file workaround needs approval.** Putting the curl in a script
   (`bash fetch.sh`) sidesteps the expansion block, but the harness requires
   interactive approval for `bash <script>` and that approval is not granted in
   automated runs (same limitation that forces `./notify` to fall back to
   `.pending-notify/`).

WebFetch cannot substitute — it cannot send the `CG-API-KEY` header. There is no
`scripts/prefetch-*.sh` case for Coinglass, so there is no cached fallback.

The skill's "Sandbox note" (point 1) assumes inline expansion "should work in most
cases" — that assumption does not hold in this harness.

## Impact

- `perps-scan` produces only the "scan unavailable" one-line artifact every run.
- `perps-brief` (downstream consumer of `.outputs/perps-scan.md`) has no regime data.
- This is the first run of the Coinglass-migrated skill — prior 2026-05-18 failure
  was a different cause (Bybit CloudFront geo-block, pre-migration).

## Recommended fix

Apply the established Aeon prefetch pattern for authenticated APIs (mirrors the
`vercel-projects` case in `scripts/prefetch-xai.sh`):

1. Add a Coinglass prefetch step (a `perps-scan` case in `scripts/prefetch-xai.sh`,
   or a dedicated `scripts/prefetch-perps.sh`) that runs before Claude with full env
   access, fetches the `coins-markets` universe + per-coin history endpoints, and
   caches the JSON responses (e.g. to `.xai-cache/perps-*.json`).
2. Modify `skills/perps-scan/SKILL.md` to read cached JSON from the prefetch
   directory instead of calling curl inline; keep the graceful-failure path for when
   the cache is absent/stale.
3. Update the skill's "Sandbox note" to document the prefetch dependency.

This requires a code change on a branch + PR (cannot be done from an automated skill
run that lacks git/PR permissions).
