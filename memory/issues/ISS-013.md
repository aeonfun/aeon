## Reproduction

1. From the GHA Claude sandbox, attempt the SKILL.md step 1 curl:
   ```
   curl -s -X POST "https://api.x.ai/v1/responses" \
     -H "Authorization: Bearer $XAI_API_KEY" \
     -d '{...}'
   ```
   → bash strips `$XAI_API_KEY` to empty before curl runs; request 401s.
2. `node -e 'console.log((process.env.XAI_API_KEY||"").length)'` → `0` (var also
   absent from `process.env`, not just bash-filtered).
3. `ls .xai-cache/` → `No such file or directory`.
4. `grep -n "reply-maker" scripts/prefetch-xai.sh` → no matches.
5. Read 2 days of `memory/logs/` for tweet-roundup / list-digest outputs → newest X
   URL status IDs decode to April 18-23 (Twitter snowflake epoch + offset math).
6. WebSearch `site:x.com polymarket OR kalshi April 26 2026` and four similar
   queries → all returned X URLs decode to April 18-23.
7. WebFetch `https://x.com/Polymarket` → HTTP 402.

## What needs to happen

**Option A — add reply-maker case to scripts/prefetch-xai.sh (recommended).**
Model on the existing `tweet-roundup` and `narrative-tracker` cases. Concrete shape:

- Read `var` from environment (passed in via the chain runner / workflow_dispatch).
- If `var` matches a numeric list ID: call X.AI `https://api.x.ai/v1/responses`
  with `tools: [{type: x_search, from_date, to_date}]` scoped to the list. Same
  POST body the SKILL.md describes, just executed pre-Claude where env vars work.
- If `var` looks like a `@handle`: same call scoped to the handle's recent posts.
- If `var` is a topic or empty: same call with `var` (or a comma-joined fallback
  pulled from MEMORY.md mission topics — Polymarket, Kalshi, Hyperliquid HIP-3,
  prediction-market calibration, agentic finance, multi-agent RL) as the search
  query.
- Write the response JSON to `.xai-cache/reply-maker-${date}-${hour}.json`.
- Skill's step 1 reads the cache file (newest matching glob) instead of curling
  X.AI directly.

This single change closes ISS-013 and unblocks every future reply-maker run.

**Option B — populate `var:` on `reply-maker` in `aeon.yml`** with a numeric list ID
the operator already curates on X. Even with Option A, the skill is more useful when
`var` points at a tight list (e.g. PM/Kalshi traders, or Hyperliquid liquidity
providers) than when it falls back to broad topic search. List IDs are the numeric
path segment in `https://x.com/i/lists/<ID>`.

**Option C — disable the cron entry** for reply-maker until Option A ships, to stop
the recurring `REPLY_MAKER_EMPTY` notification noise. Cleanest short-term fix if
Option A is not landing this week.
