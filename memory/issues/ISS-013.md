## What happened

Between 2026-04-26T23:53Z and 2026-04-26T23:58Z UTC, the entire 18:00–19:00 UTC
("evening / meta") cron slot plus all skills already mid-flight hit an
identical failure mode: workflows ran, but the Claude SDK call returned
`total_cost_usd:0` with all token counters zero. No model was invoked.
The truncated error tail recorded in cron-state for every affected skill
is the same shape:

```
"total_cost_usd":0,"usage":{"input_tokens":0,"cache_creation_input_tokens":0,
"cache_read_input_tokens":0,"output_tokens":0,
"server_tool_use":{"web_search_requests":0,"web_fetch_requests":0}
```

53 enabled skills are now in `CRITICAL` status (consecutive_failures >= 3).
Six skills that re-ran in the 02:00–02:15Z window today (2026-04-27)
recovered cleanly (token-alert, on-chain-monitor, defi-monitor, treasury-info,
token-report, auto-merge — all DEGRADED only because historical success_rate
is dragged down by the prior failure burst). This rules out a permanent
key/quota outage and points at a transient infra issue.

## Signal

- 53 skills CRITICAL, all sharing the zero-token last_error signature.
- Mass-fail window is tight: 23:53Z → 23:58Z (5 minutes), matches one
  cron tick where the evening slot fans out (~80 jobs).
- 6 skills that hit the next scheduled fire time post-window succeeded.
- Heartbeat itself is in the affected set (cf=14, last_success
  2026-04-26T08:09:17Z, last_failed 2026-04-26T23:56:13Z).
- Not the same root cause as ISS-001/002/012 (sandbox-blocked specific
  hosts) or ISS-003..011 (skill-evals no_file_match / key mismatches).

## What needs to happen

1. Pull GHA logs for any one of the 53 failing runs around 23:53–58Z on
   2026-04-26 (e.g. `heartbeat` run that wrote `last_failed:
   2026-04-26T23:56:13Z`). Look for: ANTHROPIC_API_KEY auth failure,
   gateway 5xx, runner DNS/network blip, or workflow YAML regression.
2. If logs show a transient API/gateway error, no code fix needed —
   skill-repair can re-dispatch the affected skills to clear their cf
   counters. Otherwise, file the underlying infra fix.
3. Once the upstream fix lands and a smoke run of 3+ affected skills
   succeeds in one slot, this issue closes via skill-health auto-resolve.
