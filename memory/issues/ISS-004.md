# ISS-004 — perps-scan Summary-blob artifact recurs after the ISS-003 guardrail

## Summary

ISS-003 tracked chain skills writing the assistant `## Summary` block into
`.outputs/{skill}.md` instead of the locked-format artifact. It was marked
resolved on 2026-05-19 after a prose guardrail was added to the perps-scan and
narrative-tracker SKILL.md files.

On the same day, the morning-review chain ran and perps-scan again wrote the
`## Summary` blob into `.outputs/perps-scan.md`. The artifact for 2026-05-19
begins with `## Summary` and carries the `**What I did**` / `**Result**` /
`**Files**` / `**Follow-ups**` structure — not the v3 locked format.

ISS-003's own resolution note anticipated this exact case: "If the bug recurs,
the guardrail prose isn't strong enough and we'll need a structural fix."

## Impact

- Low-to-medium. The Summary blob carries the headline verdict (QUIET, 1
  ACCUMULATION) and the WATCH bucket, so downstream perps-brief still consumed
  the perps-scan read and produced its brief.
- The v3 locked per-asset raw metric table is lost for any consumer that needs
  more than the headline, and for cross-day audit skills reading historical
  artifacts.

## Recommended fix

The prose guardrail added under ISS-003 is insufficient. A structural fix is
needed, as ISS-003's resolution note flagged:

- Write the artifact via a templated command (e.g. a jq-built or heredoc
  string composed before any `## Summary` text exists in context), or
- Split the artifact write into a separate sub-skill or step whose context
  contains no assistant `## Summary` block at all.

narrative-tracker shares the same dual-output pattern and should be patched the
same way even though it produced a clean artifact this run.

## Workaround

`.outputs/perps-scan.md` for 2026-05-19 holds the Summary blob. A perps-scan
re-run regenerates it in the correct format. daily-ops-review is consume-only
and does not re-run it.

## Recurrence — 2026-05-19 afternoon corrective re-run

The afternoon morning-review chain re-ran perps-scan explicitly to fix this
bug. The re-run's own assistant summary states the artifact was "rewritten in
the v3 locked format, overwriting the malformed `## Summary` blob". It was not.
`.outputs/perps-scan.md` still opens with a prose line plus `## Summary` /
`**What I did**` / `**Result**` / `**Files**` / `**Follow-ups**` — the blob,
not the v3 locked format with the per-asset raw metric table.

This is the third occurrence in one day (original chain run, then a deliberate
corrective re-run) and confirms ISS-004's central point: a re-run with explicit
intent to produce the locked format still produced the blob. The bug is not a
one-off lapse — the assistant cannot reliably suppress its own `## Summary`
output at artifact-write time. The structural fix (templated/heredoc artifact
write, or a separate write step with no `## Summary` in context) is required;
re-running is not a reliable workaround.

## Resolution (2026-05-19)

Shipped a structural fix per the recommendation above. The previous design — Claude writing `.outputs/perps-scan.md` directly with prose-level guardrails to keep the artifact and assistant `## Summary` separate — was abandoned.

New flow:
1. `skills/perps-scan/SKILL.md` step 12 now writes `.outputs/perps-scan.data.json`, a structured intermediate. No markdown artifact is written by the skill.
2. `scripts/render-perps-scan.py` renders `.outputs/perps-scan.md` deterministically from that JSON using a fixed template — pure Python, no LLM in the path.
3. `scripts/postprocess-perps-scan.sh` invokes the render. The workflow's `Run post-process scripts` step auto-loops `scripts/postprocess-*.sh`, so wiring is automatic (gated on JSON presence — safe no-op for other skills).
4. On malformed JSON the render writes a `scan unavailable, render failed` placeholder and exits non-zero so `daily-ops-review` surfaces the failure.

Because Claude no longer writes the markdown artifact, the `## Summary` blob cannot land there — there is no path for it. Closes the recurrence vector at the architecture level.

Files in fix:
- `skills/perps-scan/SKILL.md` (step 12 rewritten; ISS-003 prose guardrail removed)
- `scripts/render-perps-scan.py` (new, ~210 lines)
- `scripts/postprocess-perps-scan.sh` (new, ~15-line shim invoking the renderer)

Validation done locally:
- Smoke test of a representative fixture renders correctly into the v3 locked format
- Malformed JSON → exits 2 with fallback placeholder
- Missing JSON → exits 0 silently (no-op for non-perps-scan runs)
- Missing required key → exits 2 with clear error
- `prefetch_failed` edge case → renders the one-line variant
